# Planejamento e Próximos Passos no IDE (Codespaces)

## Objetivo Geral
Avançar no desenvolvimento do artigo científico, executando as tarefas diretamente no IDE fornecido pelo GitHub Codespaces para permitir maior agilidade e organização no processo de escrita, pesquisa e estruturação de seções.

---

## Estrutura no Codespaces

Ao trabalhar no Codespaces, recomendamos manter os seguintes diretórios e arquivos organizados:

### Diretórios
- `introducao/`: Contém os arquivos relacionados à seção de introdução e conceitos fundamentais.
- `engenharia-de-prompts/`: Estruturas sobre boas práticas e engenharia de *prompts*.
- `contextos/`: Materiais de apoio e exemplos práticos sobre o trabalho com contextos.
- `uso-pratico/`: Exemplos, tutoriais e *walkthroughs* para aplicação prática da IA no desenvolvimento de software.
- `comparativos/`: Dados, gráficos e análises comparativas entre diferentes modelos de IA.
- `referencias/`: Repositório de citações, *whitepapers*, e fontes utilizadas.

### Arquivos
- `todo-desenvolvimento-artigo.md`: Acompanhamento de tarefas principais.
- Novo documento para cada seção, como:
  - `introducao.md`
  - `engenharia-de-prompts.md`
  - `contextos.md`
  - `uso-pratico.md`
  - `comparativos.md`

---

## Tarefas no IDE

### 1. Criar Arquivos e Estruturar Seções
- [ ] Iniciar a seção **Introdução** no arquivo `introducao/introducao.md`.
- [ ] Estruturar os pontos principais sobre **engenharia de prompts** no diretório `engenharia-de-prompts/`.
- [ ] Adicionar exemplos práticos no diretório `uso-pratico/`.
- [ ] Levantar métricas e análise de modelos no diretório `comparativos/`.

### 2. Colaborar no Desenvolvimento do Artigo
- [ ] Dividir responsabilidades e definir subtarefas usando o `todo-desenvolvimento-artigo.md`.
- [ ] Compartilhar atualizações de progresso no repositório para facilitar integração das mudanças.

### 3. Criar Configurações no IDE para Produtividade
- [ ] Definir *extensions* úteis como editor de Markdown e suporte para pré-visualização.
- [ ] Configurar ambientes para análise de código reproduzível (se for necessário para exemplos de IA em códigos).

---

## Orientações Gerais

### Sobre o Desenvolvimento das Seções
- Certifique-se de manter as seções fáceis de entender e bem referenciadas.
- Use exemplos práticos sempre que possível para suportar a explicação teórica.
- Crie gráficos ou tabelas de apoio no diretório `comparativos/`.

### Sobre o Repositório
- Utilize *commits* descritivos para manter o histórico claro.
- Evite trabalhar diretamente no mesmo arquivo em paralelo — coordene-se com colaboradores.
